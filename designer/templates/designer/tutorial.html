How this tool works

This tutorial explains the biology behind CRISPR base editing and walks you through how to use this designer step by step.

1. What is CRISPR base editing?

Classic CRISPR-Cas9 cuts both strands of DNA. Base editors are more surgical: instead of cutting, they chemically convert one DNA base into another, directly at the target site, without breaking the double helix.

This makes base editors ideal for studying or correcting point mutations which are the cause of many inherited diseases.

Source: Original

Both editors, the cytosine base editor (CBE) and the adenine base editor (ABE), are fused to a Cas9 nickase that is guided to the exact genomic location by a guide RNA. The deaminase enzyme then converts the target base within a small editing window — typically positions 4–8 of the 20-nt protospacer.

2. The editing window

Not every base in the protospacer can be edited. The deaminase only reaches bases that fall within a defined editing window, counted from the 5′ end of the guide RNA (position 1) toward the PAM (position 20).

Source: Original

A guide RNA is only useful for base editing if the target base (A for ABE, C for CBE) falls within this window and a suitable NGG PAM sequence exists nearby. This is one reason not every amino acid position can be targeted.

3. Why AlphaMissense?

Not all amino acid changes are clinically relevant. AlphaMissense is a deep-learning model from Google DeepMind that predicts the pathogenicity of missense variants — single amino acid substitutions — on a scale from 0 (benign) to 1 (likely pathogenic).

This tool uses AlphaMissense scores to rank guide RNAs: positions where the base edit would introduce a highly pathogenic variant are shown first. This helps you focus your experiments on the most disease-relevant targets.

4. How to use this tool

1

Enter a UniProt or Ensembl ID

Find your protein of interest on uniprot.org and copy the accession (e.g. P04637 for TP53). You can also enter an Ensembl transcript ID starting with ENST. The tool will automatically retrieve the coding sequence.

2

Set the AlphaMissense threshold

Only positions where at least one possible base edit leads to a variant with an AlphaMissense score above your threshold will be shown. The default is 0.6 — a good starting point that filters out likely benign variants while keeping most pathogenic candidates.

3

Choose your editor

Select ABE for A→G edits, CBE for C→T edits, or ABE & CBE to see candidates for both. The tool will only show positions where the codon contains an editable base (A for ABE, C for CBE) and a compatible guide RNA can be designed.

4

Choose your PAM sequence

NGG is the standard SpCas9 PAM. However, if you prefer you can also select NG to find more guide RNA candidates or NAG.

5

Guide RNAs at multiple positions

A single guide RNA sequence can sometimes edit more than one amino acid position. Only uniquely mapping guides hides guides that map to multiple positions. Best position per guide keeps one row per guide (highest AlphaMissense score) and shows a ×N badge when that guide maps to N positions. All occurrences, grouped lists every position; identical guides are sorted together. Row colours appear only when two or more rows with the same guide are visible (filters may hide partners).

6

Interpret the results

Results are sorted by AlphaMissense score, highest first. Each row shows one guide RNA for one amino acid position. The Guide RNA Outcomes column lists all possible amino acid changes this guide could produce, with pathogenic variants marked with * and their score in brackets. Use this to identify guides with clean, specific edits.

7

Download your results

Use the Download CSV or Download Excel buttons on the results page to export your guide RNA candidates for further analysis or ordering.
